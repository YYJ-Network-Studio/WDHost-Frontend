DNS無法使用的特徵以及解決辦法

台灣VPS用戶注意

曾有幾位用戶回報DNS無法使用，請先嘗試重啟主機，有部分用戶重啟後即可解決。

常見DNS錯誤造成無法使用網路的特徵

使用apt install顯示Temporary failure resolving 'tw.archive.ubuntu.com'

使用ping google.com顯示ping: Temporary failure in name resolution

給新手的小知識：DNS無法使用，代表無法解析域名(Domain)，假如Ping的是IP，依然可以正常運作。

如何解決

輸入resolvectl status

Current DNS Server:代表目前使用的DNS Server

DNS Servers:第一個IP代表主要DNS Server，第二個代表次要DNS Server

推薦使用Google的DNS，主要DNS：8.8.8.8，次要DNS：8.8.4.4

使用Netplan更改DNS Server

輸入nano /etc/netplan/00-installer-config.yaml編輯檔案

在Interface(例如：eth0, enp7s0, ens18)裡面新增nameservers(請注意縮排)

nameservers:
(這裡要空4格)address: [8.8.8.8, 8.8.4.4]

儲存修改後，輸入netplan apply就完成DNS更改了。
